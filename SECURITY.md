# Security Policy

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Use GitHub's private disclosure flow instead:
**[Report a vulnerability](https://github.com/heindewilde/someday/security/advisories/new)**

Include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- The version or commit you tested against
- Any suggested fix, if you have one

You'll receive a response within **72 hours**. If the issue is confirmed, a fix will be prioritised and a GitHub Security Advisory published once it's resolved. Credit is given to reporters unless you prefer to stay anonymous.

## Scope

Someday is a self-hosted app — each instance is operated independently. Reports are most valuable for vulnerabilities in the application code itself (auth, data isolation, input handling, dependency issues) rather than in a specific deployment's configuration.
